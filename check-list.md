# Чек-лист ревизии проекта FastAPI

## Реализовано
- Аутентификация, регистрация, JWT access/refresh токены, валидация пароля.
- Импорт/экспорт/поиск офферов, фильтрация, работа с БД через репозиторий.
- Асинхронные endpoint-ы, Pydantic v2 схемы, разделение сервисов и моделей.

## Не реализовано/проблемы
- Нет endpoint для refresh токена (обновления access).
- Нет защищённых endpoint-ов (offers доступны без авторизации).
- Нет logout/revoke refresh токена.
- Нет тестов, логирования, обработки ошибок на уровне API.
- Нет CORS, CSRF, rate-limit, RBAC.
- Нет OpenAPI описаний, нет валидации уникальности email на уровне БД.
- Нет rollback транзакций при ошибках.

## Потенциальные баги/риски
- Любой может импортировать/экспортировать офферы.
- Возможны silent ошибки при изменении структуры данных.
- Нет защиты от brute-force, нет проверки CORS/CSRF.

## Best practices
- Используется Pydantic v2, асинхронность, разделение слоёв.
- Не используются Depends(get_current_user) для защиты, не все endpoint-ы с response_model.

## Рекомендации
- Добавить refresh endpoint, revoke/logout.
- Защитить offers через Depends(get_current_user), реализовать RBAC.
- Добавить api_response, обработку ошибок, логирование, тесты, OpenAPI, CORS, CSRF, миграции, валидацию уникальности.
- Вынести секреты в .env.

## Оценка готовности
- Базовая архитектура есть, но для продакшена нужно доработать безопасность, обработку ошибок, тесты и документацию.
- Готовность: около 60% (MVP, не production-ready).
